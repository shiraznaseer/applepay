Tamara - API Payloads

Tamara endpoints and sample payloads implemented in this project.

Base URL

Production example: https://applepay.tamarran.com
Webhook endpoint: POST /api/tamara/webhook

Webhook auth

Webhook token is expected in either:
  • Authorization: Bearer <tamara_notification_jwt>
  • POST /api/tamara/webhook?tamaraToken=<tamara_notification_jwt>
Config used for validation:
  • Tamara:NotificationToken (HS256 secret)
  • Tamara:NotificationIssuer (optional issuer validation)

Tamara APIs

POST /api/tamara/checkout
Note: Forwards the JSON body to Tamara POST /checkout. If config Tamara:NotificationUrl is set, the server auto-injects merchant_url.notification when missing.

POST /api/tamara/webhook
Behavior:
  • Logs to Logs/tamara-webhook.log
  • Saves order + webhook event history to SQL if Tamara:DbConnectionString is configured
  • Broadcasts WebSocket event tamara.order.updated
  • On order_approved event, calls authorise then capture server-to-server.

Request headers (option 1)

Authorization: Bearer <tamara_notification_jwt>
Content-Type: application/json

Request (option 2)

POST /api/tamara/webhook?tamaraToken=<tamara_notification_jwt>

Request body (example)

{
  "event_type": "order_approved",
  "order_id": "TAMARA-ORDER-ID",
  "status": "approved"
}

Response

{
  "received": true
}
